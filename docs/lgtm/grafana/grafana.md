# Grafana

[Grafana](https://grafana.com/grafana/) is the visualisation and dashboarding layer of the LGTM stack. It connects to Loki for logs and Prometheus for metrics, and is protected by Authentik SSO.

## Prerequisites

- `lgtm` namespace exists
- cert-manager and Traefik are running
- Authentik is configured and the OAuth2 provider and application are set up for Grafana (see below)

## Install

```bash
helm repo add grafana https://grafana.github.io/helm-charts
helm repo update
kubectl apply -f certificate.yaml
kubectl apply -f grafana-oauth-secret.yaml
helm install grafana grafana/grafana --namespace lgtm -f grafana-values.yaml
```

`grafana-oauth-secret.yaml` contains the Authentik OAuth2 client credentials and is not committed to the repository — retrieve it from the password vault.

All available Helm values are in the [Grafana Helm chart README](https://github.com/grafana/helm-charts/blob/main/charts/grafana/README.md).

Verify the pod is running:

```bash
kubectl get pods -n lgtm -l app.kubernetes.io/name=grafana
```

Test the endpoint:

```bash
curl -kv https://grafana.local.spaelling.xyz/
```

## Initial admin password

The Helm chart generates a random admin password on first install. Retrieve it to make other users admin before SSO is working:

```bash
kubectl get secret --namespace lgtm grafana -o jsonpath="{.data.admin-password}" | base64 --decode ; echo
```

Log in as `admin` with this password, promote your Authentik-linked user to admin, then you can rely on SSO going forward.

## Upgrade

```bash
helm upgrade grafana grafana/grafana --namespace lgtm -f grafana-values.yaml
```

If pods get stuck after an upgrade (new pods in `Pending` or `Error` while old pods are still running), delete the stuck pods — Kubernetes will reschedule them:

```bash
kubectl delete pod -n lgtm -l app.kubernetes.io/name=grafana
```

## Authentik integration

Follow the [Authentik Grafana integration guide](https://docs.goauthentik.io/integrations/services/grafana/) to create the OAuth2 provider and application in Authentik.

The `grafana-values.yaml` in this directory already includes the `grafana.ini` settings for Authentik OAuth2. The only thing needed is the `grafana-oauth-secret.yaml` containing the client ID and secret generated by Authentik.
